PyFence

Smart type safety for Python

  • lets you verify type compliance when debugging.
  • doesn't mess with your code.
  • extracts type information from Sphinx docstrings.

Problematic code


class RationalFormatter (object):
    def format(self, number):
        """
        Stringifies a number to numerator/denominator format
        Example::

            >>> print(RationalFormatter().format(1.25))
            5/4

        :param number: input number
        :type  number: float
        :raises      : None
        :rtype       : str
        """
        return '%i/%i' % number.as_integer_ratio()

    def display(self, number):
        print(str(number) + ' = ' + self.format(number))
                            

from example_formatter import RationalFormatter

f = RationalFormatter()
f.display(1.25)
f.display(5)  # boom

Writing docstrings

PyFence parses Sphinx-compatible docstrings, so, along with type verification, you also get nice autogenerated documentation!
Example docstring: 


class RationalFormatter (object):
    def format(self, number):
        """
        Stringifies a number to numerator/denominator format

        :param          number: input number
        :type           number: float
        :param bool flag_param: inline type specification
        :raises: None
        :rtype: str
        """

PyFence recognized following Sphinx tags:

  • :type argname: typelist - allowed argument types
  • :rtype: typelist - allowed return types
  • :raises: typelist - allowed exception types

Type specification

  • str - builtin type name
  • ajenti.ui.UIElement - fully qualified type name
  • UIElement - nonqualified type name
  • bool, str, int - multiple allowed types
  • None - NoneType
  • any - wildcard
  • class - new-style classes (== types.TypeType)
  • function - functions and methods

Running from CLI

fence <python file> [arguments] --fence:param1,param2=value --fence:param3

Parameters

stop

Stop after first error. 

$ fence example.py --fence:stop
1.25 = 5/4
 *** PyFence ERROR ---------------------------
 *** PyFence ERROR PyFence verification failed
 *** PyFence ERROR  :: example_formatter.RationalFormatter.format(<example_formatter.RationalFormatter object at 0x7fa47cc8ec10>, 5)
 *** PyFence ERROR in /home/eugeny/Work/pyfence/example_formatter.py:2
 *** PyFence ERROR number was 5 (int) instead of float
 *** PyFence ERROR Aborting

strict

Strict mode: types must match exactly, e.g. 'file' won't match 'object'

include/exclude

Indicate which modules will be fenced. Regexes supported. $ fence app.py --fence:include=app,exclude=app.lib

lint

Report parameters with missing type spec. 

$ fence example.py --fence:lint
 *** PyFence WARN  example_formatter.RationalFormatter.format
 *** PyFence WARN  No type specified for argument 'new_arg'

debug

Include debugging output from Fence. 

$ fence example.py --fence:debug,include=example
